태그 읽기 — 표시위치: 상단(영업이익 안 · 조정 가능) / 하단·불명(참고). · 손익계산서 확정: XBRL 손익계산서 라인과 당기금액이 일치해 결정론으로 확정(계정명 매칭 아님). · 근거 확인: LLM 이 제시한 계상 위치 근거가 인용에 실제로 있음이 검증됨(없으면 상단→불명 강등 = 근거 게이트). · ⚠ 표시위치 불안정: 반복 실행에서 회차마다 달라 조정에서 제외(원문 확인). · 근거강도: 명시(금액이 인용에 그대로) / 추정(표 구조에서 유추 — 원문보기 발췌를 넓혀 근거를 함께 보임). · 조정에 포함(수동): 불명 항목을 원문 확인 후 사용자가 직접 조정에 넣는 스위치(도구 판단 아님). · 합계·구성 계층(1단계·산술): 주석 표의 산술(합계=구성 합)로 증명한 포함관계 → 잠금. 합계를 넣으면 구성 일괄 반영(구성 잠김), 구성만 개별 선택도 가능 — 이중가산 산술 불가. 이름 못 얻은 나머지는 그 외(도구 계산값)로 채워 합=구성을 맞춘다. · 추정 묶음(2단계·근접): 서로 다른 주석이라 산술은 못 했으나 금액이 근접(≥99.5%)·동부호·동성격이라 포함관계로 추정해 잠근 것 — 추정임을 밝히고 [묶음 해제]로 사용자가 풀 수 있다(기본은 안전측 잠금). · ⚠ 포함관계 가능성(3단계·경고): 근접 90~99.5% — 남남 가능성이 있어 잠그지 않고 경고만(원문 확인 요망). · D&A 이미 포함: 후보 금액이 D&A(감가·무형 상각) 셀(XBRL 개념코드로 식별)과 일치하면 조정 자격을 박탈해 참고로 — 조정에 넣으면 EBITDA 에 이중 반영되기 때문. 손상차손 등은 D&A 개념이 아니라 안 걸린다(과제거 방지).

구역 A — EBITDA 브릿지

구역 B — 조정 대상 (영업이익 위 · 일회성)

영업이익 안에 이미 반영된 일회성 항목입니다. 체크하면 EBITDA 에서 제외합니다 — 이익 항목은 빼고, 비용 항목은 도로 더합니다. (매출원가·판관비에 묻힌 일회성을 캐내는 자리)

구역 C — 참고 항목 (영업이익 아래 · 표시위치 불명)

영업이익 아래(기타영업외 등)에 표시됐거나 표시위치가 불명한 항목입니다. 애초에 EBITDA 에 없어 조정(도로 더하기) 대상이 아닙니다 — 손익계산서에 별도 줄로 보이니 원문으로 확인만.
금액 기준(백만원 이상만 표시):
표시위치 불명·불안정 — 영업이익 위/아래를 확정 못 했거나 회차마다 달라 도구는 조정하지 않음.
도구가 계상 위치를 확인하지 못한 항목입니다. 원문을 확인하고 영업이익 안에 계상된 것이 맞으면 아래 “조정에 포함(수동)” 스위치로 직접 포함하십시오 — 손익방향(이익=제외 시 뺌 / 비용=제외 시 더함)을 정하면 EBITDA 에 반영됩니다. 수동 포함은 사용자 판단이며 도구 판단과 구분됩니다.
금액 없는 정성적 항목 — 우발부채·보고기간후사건 등. 금액이 없어 슬라이더 대상이 아닙니다.
