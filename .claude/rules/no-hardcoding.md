# No Hardcoding

## 원칙
판단을 지배하는 마법 상수를 코드에 박지 않는다. 특정 회사·업종에 맞춘 값도 금지.

## 특히 screen에서
괴리 경고를 "비율 < 0.8이면 빨간불" 같은 임계 상수로 만들지 마라.
괴리의 크기와 지속연수를 등급(gradient)으로 노출한다. 판정선은 사람이 본다.

## 나쁨 / 좋음
나쁨: if cfo/op < 0.8: flag = RED
좋음: 3~5년 누적 괴리율과 지속연수를 그대로 등급으로 반환

## 예외
회계기준이 정한 구조적 상수(예: 표준 주석 섹션 위치)는 하드코딩이 아니다. 그건 보편 구조다.
