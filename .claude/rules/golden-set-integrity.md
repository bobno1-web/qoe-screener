# Golden Set Integrity

## 원칙
정답지는 도구가 만들지 않는다. 도구 출력이 자기 채점 기준이 되면 하네스가 자기 맹점을 못 본다.

## 물리적 분리 (폴더로 이미 강제, 여기서 명문화)
- out/ 의 어떤 파일도 golden/ 으로 들어오지 않는다.
- 채점 기준은 오직 golden/ratified/ (사람 승인분)뿐이다. drafts/ 는 정답 아님.
- golden/draft-prompt/ 는 도구 프롬프트(prompts/)와 절대 병합하지 않는다. 초안은 도구보다 일부러 넓게(과잉포함) 던진다.

## 왜 초안 프롬프트가 도구와 달라야 하나
같은 프롬프트로 초안을 뽑고 사람이 도장만 찍으면, 도구와 정답지가 같은 실수를 공유한다. 초안은 도구가 놓칠 것까지 사람 앞에 올려야 한다.

## 시드
회사가 IR에서 자체 공시한 조정 실적 브릿지는 순환 없는 공짜 정답이다. seed-ir/ 에 담아 출발점으로 쓴다.

## 향후
out/→golden/ 복사 차단은 훅으로 자동 강제한다(다음 단계).
