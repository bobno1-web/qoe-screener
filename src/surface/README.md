감사보고서와 주석을 LLM으로 읽어 일회성 의심 후보를 인용과 함께 올리는 발굴기. 1-a/1-b를 별도 폴더로 쪼개지 않는다 — 이 폴더 하나로 둔다. 최종 일회성 판정은 넣지 않는다(감사인 몫). 프롬프트는 prompts/에 둔다.

핵심 가치는 하단(EBITDA 밖) 회색지대 발굴이다 — 상단(명백한 비반복)은 눈에 띄어 보조적. 도구는 반복/비영업을 스스로 판정해 후보에서 거르지 않는다("비영업이라 뺀다" 금지). 넓게 올리고 판정은 감사인이 한다. 각 후보에 성격(비반복 / 비영업·비현금 / 추정품질, 복수 가능) + EBITDA구분(상단/하단/불명) 태그를 붙여 상단·하단을 자동 분류한다. 리콜 우선 — 넓히면 정밀도가 떨어지는 건 정상이고 슬라이더·사람이 감당한다. 기준선 정의는 CLAUDE.md.
